* URL이 PHP로 끝남.
    * 언어가 PHP 라는 것을 알려주는 것이고,
    * 레거시 프로젝트임을 암시하는 것.
    * 공격자가 PHP언어에 맞는 공격법을 사용할 수 있다.

* HTTP METHOD 가 용도에 맞지 않게 사용되었음
    * GET, POST, PUT, PATCH, DELETE, HEAD, TRACE, OPTIONS
* 로깅과 테스트가 없다.
    * 에러를 추적하기 어렵다
    * 하나를 수정하면 다른 기능이 정상적으로 동작하는 보장이 없다. (테스트)
* 비즈니스 로직, 뷰가 합쳐져 있다.
    * 소스코드 노출될 경우, 해킹위험 크다
    * 유지보수가 어렵다
* 도큐먼트 루트에 여러문서가 섞여 있다.
    * 공격자가 프로젝트에 접근 할 수 있다.
    * 파일 다운로드 & 파일 업로드 취약점이 생길 수 있다.
* 모듈화가 되지 않았음.
    * 함수화를 통해서 하는 게 좋다.
    * 유지보수가 어렵다.
